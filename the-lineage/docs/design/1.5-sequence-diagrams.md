# 1.5 — Sequence Diagrams

## 5.1 — POST /listings (Seller creates a listing)

```mermaid
sequenceDiagram
    autonumber
    actor Seller
    participant LC as ListingController
    participant JWT as JwtAuthFilter
    participant LS as ListingService
    participant SS as SellerProfileService
    participant SHS as ShoeService
    participant PS as ProvenanceService
    participant LR as ListingRepository
    participant SHR as ShoeRepository
    participant PR as ProvenanceRepository
    participant DB as PostgreSQL

    Seller->>LC: POST /listings (Bearer JWT, body)
    LC->>JWT: validate token
    JWT-->>LC: principal(UUID, role=SELLER)
    LC->>LS: create(sellerUserId, request)
    LS->>SS: findApprovedProfile(sellerUserId)
    SS->>DB: SELECT seller_profile
    DB-->>SS: profile (APPROVED)
    SS-->>LS: profile
    LS->>SHS: findById(shoeId)
    SHS->>SHR: findById
    SHR->>DB: SELECT shoe
    DB-->>SHR: shoe
    SHR-->>SHS: shoe
    SHS-->>LS: shoe (authenticated)
    LS->>LS: assert shoe.seller == profile
    LS->>LS: assert shoe.authenticated
    LS->>LR: save(new Listing, state=AVAILABLE)
    LR->>DB: INSERT listing
    DB-->>LR: listing
    LR-->>LS: listing
    LS->>PS: append(shoeId, sellerUserId, LISTED, payload)
    PS->>PR: save(new ProvenanceRecord)
    PR->>DB: INSERT provenance_record
    DB-->>PR: record
    PR-->>PS: record
    PS-->>LS: record
    LS-->>LC: listing
    LC-->>Seller: 201 Created + ListingDto
```

## 5.2 — POST /checkout (Buyer completes purchase)

```mermaid
sequenceDiagram
    autonumber
    actor Buyer
    participant CO as CheckoutController
    participant JWT as JwtAuthFilter
    participant CS as CartService
    participant LS as ListingService
    participant PS as PaymentService
    participant OS as OrderService
    participant PRS as ProvenanceService
    participant NS as NotificationService
    participant DB as PostgreSQL

    Buyer->>CO: POST /checkout (Bearer JWT)
    CO->>JWT: validate token
    JWT-->>CO: principal(UUID, role=BUYER)
    CO->>CS: getActiveCart(buyerId)
    CS->>DB: SELECT cart + items
    DB-->>CS: cart (1 item, listing=L)
    CS-->>CO: cart
    CO->>LS: reserve(listingId)
    LS->>DB: SELECT listing FOR UPDATE
    DB-->>LS: listing AVAILABLE
    LS->>DB: UPDATE listing SET state=RESERVED
    DB-->>LS: ok
    LS-->>CO: reserved
    CO->>OS: createFromCart(buyerId)
    OS->>DB: INSERT order(status=CREATED)
    DB-->>OS: order
    OS-->>CO: order
    CO->>PS: initiate(order.id)
    PS->>DB: INSERT payment(status=INITIATED)
    PS->>PS: call external processor (mock)
    PS->>DB: UPDATE payment(status=CAPTURED, escrow=HELD)
    DB-->>PS: payment
    PS-->>CO: payment
    CO->>LS: markSold(listingId)
    LS->>DB: UPDATE listing SET state=SOLD
    DB-->>LS: ok
    LS->>PRS: append(shoeId, buyerId, SOLD, payload)
    PRS->>DB: INSERT provenance_record
    DB-->>PRS: record
    CO->>OS: markPaid(order.id)
    OS->>DB: UPDATE order SET status=PAID
    CO->>NS: orderPlaced(buyerId, sellerId, orderId)
    NS-->>CO: queued
    CO-->>Buyer: 200 OK + OrderDto
```

## 5.3 — POST /sellers/applications

```mermaid
sequenceDiagram
    autonumber
    actor User
    participant SAC as SellerApplicationController
    participant JWT as JwtAuthFilter
    participant SAS as SellerApplicationService
    participant SAR as SellerApplicationRepository
    participant NS as NotificationService
    participant UR as UserRepository
    participant DB as PostgreSQL

    User->>SAC: POST /sellers/applications (JWT, body)
    SAC->>JWT: validate
    JWT-->>SAC: principal(UUID, role=BUYER)
    SAC->>SAC: bean validation on request
    SAC->>SAS: submit(userId, applicationData)
    SAS->>SAR: findPendingByUser(userId)
    SAR->>DB: SELECT
    DB-->>SAR: empty
    SAR-->>SAS: none
    SAS->>SAR: save(application, status=PENDING)
    SAR->>DB: INSERT seller_application
    DB-->>SAR: application
    SAR-->>SAS: application
    SAS->>UR: findAllByRole(CURATOR)
    UR->>DB: SELECT
    DB-->>UR: curators
    UR-->>SAS: curators
    SAS->>NS: notifyAll(curators, APPLICATION_SUBMITTED, payload)
    NS-->>SAS: queued
    SAS-->>SAC: application
    SAC-->>User: 201 Created + ApplicationDto
```

## 5.4 — GET /listings/{id} (Public)

```mermaid
sequenceDiagram
    autonumber
    actor Visitor
    participant LC as ListingController
    participant LS as ListingService
    participant PS as ProvenanceService
    participant SPS as SellerProfileService
    participant LR as ListingRepository
    participant PR as ProvenanceRepository
    participant SPR as SellerProfileRepository
    participant DB as PostgreSQL

    Visitor->>LC: GET /listings/{id}
    note over LC: no auth required
    LC->>LS: findById(id)
    LS->>LR: findById(id)
    LR->>DB: SELECT listing JOIN shoe
    DB-->>LR: listing + shoe
    LR-->>LS: listing
    LS-->>LC: listing
    LC->>PS: chainFor(shoe.id)
    PS->>PR: findByShoeIdOrderByOccurredAtAsc
    PR->>DB: SELECT provenance_record
    DB-->>PR: records
    PR-->>PS: records
    PS-->>LC: chain
    LC->>SPS: publicProfile(listing.seller.id)
    SPS->>SPR: findById
    SPR->>DB: SELECT seller_profile
    DB-->>SPR: profile
    SPR-->>SPS: profile
    SPS-->>LC: public projection
    LC->>LC: assemble ListingDetailDto<br/>(listing + shoe + chain + seller)
    LC-->>Visitor: 200 OK + ListingDetailDto
```
