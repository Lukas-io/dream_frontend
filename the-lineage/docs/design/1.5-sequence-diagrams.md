# 1.5 — Sequence Diagrams

## 5.1 — POST /listings (Seller creates a listing)

```mermaid
sequenceDiagram
    autonumber
    actor Seller
    participant LC as ListingController
    participant JWT as JwtAuthFilter
    participant LS as ListingServiceImpl
    participant SPR as SellerProfileRepository
    participant SHR as ShoeRepository
    participant PS as ProvenanceService
    participant LR as ListingRepository
    participant PR as ProvenanceRecordRepository
    participant DB as PostgreSQL

    Seller->>LC: POST /listings (Bearer JWT, body)
    LC->>JWT: validate token
    JWT-->>LC: principal(UUID, role=SELLER)
    LC->>LS: create(sellerUserId, request)
    LS->>SPR: findByUserId(sellerUserId)
    SPR->>DB: SELECT seller_profile
    DB-->>SPR: profile
    SPR-->>LS: profile
    LS->>LS: assert profile.applicationStatus == APPROVED
    LS->>SHR: findById(shoeId)
    SHR->>DB: SELECT shoe
    DB-->>SHR: shoe
    SHR-->>LS: shoe (authenticated)
    LS->>LS: assert shoe.seller == profile
    LS->>LS: assert shoe.isAuthenticated
    LS->>LR: save(new Listing, state=AVAILABLE)
    LR->>DB: INSERT listing
    DB-->>LR: listing
    LR-->>LS: listing
    LS->>PS: append(shoeId, sellerUserId, LISTED, payload)
    PS->>PR: save(new ProvenanceRecord)
    PR->>DB: INSERT provenance_record
    DB-->>PR: record
    PR-->>PS: record
    PS-->>LS: record
    LS-->>LC: listing
    LC-->>Seller: 201 Created + ListingDto
```

## 5.2 — POST /checkout (Buyer completes purchase)

```mermaid
sequenceDiagram
    autonumber
    actor Buyer
    participant CO as CheckoutController
    participant JWT as JwtAuthFilter
    participant CS as CartService
    participant LS as ListingService
    participant PS as PaymentService
    participant OS as OrderService
    participant PRS as ProvenanceService
    participant NS as NotificationService
    participant DB as PostgreSQL

    Buyer->>CO: POST /checkout (Bearer JWT)
    CO->>JWT: validate token
    JWT-->>CO: principal(UUID, role=BUYER)
    CO->>CS: getActiveCart(buyerId)
    CS->>DB: SELECT cart + items
    DB-->>CS: cart (1 item, listing=L)
    CS-->>CO: cart
    CO->>LS: reserve(listingId)
    LS->>DB: SELECT listing FOR UPDATE
    DB-->>LS: listing AVAILABLE
    LS->>DB: UPDATE listing SET state=RESERVED
    DB-->>LS: ok
    LS-->>CO: reserved
    CO->>OS: createFromCart(buyerId)
    OS->>DB: INSERT order(status=CREATED)
    DB-->>OS: order
    OS-->>CO: order
    CO->>PS: initiate(order.id)
    PS->>DB: INSERT payment(status=INITIATED)
    PS->>PS: call external processor (mock)
    PS->>DB: UPDATE payment(status=CAPTURED, escrow=HELD)
    DB-->>PS: payment
    PS-->>CO: payment
    CO->>LS: markSold(listingId)
    LS->>DB: UPDATE listing SET state=SOLD
    DB-->>LS: ok
    LS->>PRS: append(shoeId, buyerId, SOLD, payload)
    PRS->>DB: INSERT provenance_record
    DB-->>PRS: record
    CO->>OS: markPaid(order.id)
    OS->>DB: UPDATE order SET status=PAID
    CO->>NS: orderPlaced(buyerId, sellerId, orderId)
    NS-->>CO: queued
    CO-->>Buyer: 200 OK + OrderDto
```

## 5.4 — GET /listings/{id} (Public)

```mermaid
sequenceDiagram
    autonumber
    actor Visitor
    participant LC as ListingController
    participant LS as ListingService
    participant PS as ProvenanceService
    participant LR as ListingRepository
    participant PR as ProvenanceRecordRepository
    participant DB as PostgreSQL

    Visitor->>LC: GET /listings/{id}
    note over LC: no auth required (SecurityConfig permitAll)
    LC->>LS: findById(id)
    LS->>LR: findById(id)
    LR->>DB: SELECT listing JOIN shoe
    DB-->>LR: listing + shoe
    LR-->>LS: listing
    LS-->>LC: listing
    LC->>PS: chainFor(listing.shoe.id)
    PS->>PR: findByShoeIdOrderByOccurredAtAsc
    PR->>DB: SELECT provenance_record ORDER BY occurred_at
    DB-->>PR: records
    PR-->>PS: records
    PS-->>LC: chain
    LC->>LC: assemble ListingDetailDto<br/>(ListingDto + List&lt;ProvenanceRecordDto&gt;)
    LC-->>Visitor: 200 OK + ListingDetailDto
```

> Seller identity is not dereferenced in this response — clients that want the
> seller's public profile call `GET /sellers/{id}` (see `SellerProfileController`).

## 5.5 — POST /auth/register + POST /auth/login (entry point)

```mermaid
sequenceDiagram
    autonumber
    actor Client
    participant AC as AuthController
    participant US as UserService
    participant PE as PasswordEncoder
    participant UR as UserRepository
    participant JS as JwtService
    participant DB as PostgreSQL

    Client->>AC: POST /auth/register {email, password, displayName}
    AC->>US: register(request)
    US->>UR: existsByEmail
    UR->>DB: SELECT 1 WHERE email
    DB-->>UR: false
    UR-->>US: false
    US->>PE: encode(password)
    PE-->>US: bcrypt hash
    US->>UR: save(User role=BUYER, active=true)
    UR->>DB: INSERT users
    DB-->>UR: row
    UR-->>US: User
    US-->>AC: User
    AC-->>Client: 201 Created {id, email, role}

    Client->>AC: POST /auth/login {email, password}
    AC->>US: login(request)
    US->>UR: findByEmail
    UR->>DB: SELECT
    DB-->>UR: user row
    UR-->>US: User
    US->>PE: matches(password, hash)
    PE-->>US: true
    US->>JS: issueAccess(user)
    JS-->>US: signed JWT (15 min)
    US->>JS: issueRefresh(user)
    JS-->>US: signed JWT (7 days)
    US-->>AC: TokenPair
    AC-->>Client: 200 OK {accessToken, refreshToken}

    note over Client: Subsequent requests carry<br/>Authorization: Bearer <accessToken>
```

## 5.7 — POST /admin/disputes/{id}/resolve (admin resolves a dispute)

```mermaid
sequenceDiagram
    autonumber
    actor Admin
    participant AC as AdminController
    participant DS as DisputeService
    participant PS as PaymentService
    participant PRS as ProvenanceService
    participant DR as DisputeRepository
    participant OR as OrderRepository
    participant PR as PaymentRepository
    participant ProvR as ProvenanceRecordRepository
    participant DB as PostgreSQL

    Admin->>AC: POST /admin/disputes/{id}/resolve<br/>Bearer JWT, body {outcome, note}
    note over AC: SecurityFilterChain ensured role ADMIN
    AC->>DS: resolve(disputeId, adminId, outcome, note)
    DS->>DR: findById
    DR->>DB: SELECT dispute
    DB-->>DR: dispute (status=OPEN, order DISPUTED)
    DR-->>DS: dispute
    DS->>DS: assert outcome in {RESOLVED_BUYER, RESOLVED_SELLER}
    DS->>DS: dispute.status = outcome,<br/>resolver=admin, resolvedAt=now()
    alt outcome == RESOLVED_BUYER
        DS->>OR: save(order.status=REFUNDED)
        OR->>DB: UPDATE orders
        DS->>DR: save(dispute)
        DR->>DB: UPDATE disputes
        DS-->>AC: dispute
        AC->>PS: refund(paymentId, note)
        PS->>PR: findById
        PR->>DB: SELECT
        DB-->>PR: payment
        PR-->>PS: payment
        PS->>PR: save(paymentStatus=REFUNDED, escrowStatus=REVERSED)
        PR->>DB: UPDATE payments
    else outcome == RESOLVED_SELLER
        DS->>OR: save(order.status=COMPLETED, completedAt=now())
        OR->>DB: UPDATE orders
        DS->>DR: save(dispute)
        DR->>DB: UPDATE disputes
        DS-->>AC: dispute
        AC->>PS: releaseEscrow(paymentId)
        PS->>PR: save(escrowStatus=RELEASED, escrowReleasedAt=now())
        PR->>DB: UPDATE payments
    end
    AC->>PRS: append(shoeId, adminId, RESOLVED, payload {outcome})
    PRS->>ProvR: save(record)
    ProvR->>DB: INSERT provenance_records
    AC-->>Admin: 200 OK + DisputeDto
```

## 5.8 — Cart reservation expiry sweeper (background)

```mermaid
sequenceDiagram
    autonumber
    participant Sched as CartReservationScheduler
    participant CS as CartService
    participant CIR as CartItemRepository
    participant LR as ListingRepository
    participant DB as PostgreSQL

    note over Sched: @Scheduled fixedDelay = PT1M (every minute)
    Sched->>CS: releaseExpired()
    CS->>CIR: findAllByExpiresAtBefore(now)
    CIR->>DB: SELECT cart_items WHERE expires_at < now
    DB-->>CIR: list of expired items
    CIR-->>CS: items
    loop for each expired item
        CS->>CS: load item.listing
        alt listing.state == RESERVED
            CS->>LR: save(listing.state=AVAILABLE, stateChangedAt=now())
            LR->>DB: UPDATE listings
        end
        CS->>CIR: delete(item)
        CIR->>DB: DELETE cart_items
    end
    CS-->>Sched: count of items released
    note over Sched: Listing is back in the public catalog<br/>and can be reserved by another buyer
```
